# Definition of Done

## Before Marking Any Task Complete

- [ ] Output matches task requirements
- [ ] Stayed within `scope.md` boundaries
- [ ] Follows `conventions.md` style rules
- [ ] Updated `progress/log.md` with session summary
- [ ] Updated `progress/tasks.json` status

## Quality Criteria

### Documentation Quality

- [ ] Follows document structure template
- [ ] All required sections present
- [ ] Code examples are valid and tested
- [ ] Links are not broken

### Content Quality

- [ ] Technically accurate (matches actual API)
- [ ] Examples use realistic data
- [ ] Error scenarios documented
- [ ] No placeholder text remaining

### Formatting

- [ ] Markdown renders correctly
- [ ] Tables are properly formatted
- [ ] Code blocks have language tags
- [ ] Consistent heading hierarchy

## Common Failure Modes

| Failure | Prevention |
|---------|------------|
| Outdated examples | Verify against actual API |
| Missing error docs | Check API for all error codes |
| Broken links | Test links before completing |
| Inconsistent format | Compare against existing docs |
