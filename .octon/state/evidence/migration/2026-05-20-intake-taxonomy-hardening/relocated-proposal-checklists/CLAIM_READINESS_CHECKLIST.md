# Claim Readiness Checklist

Use this as the hard yes/no closeout sheet before any final target-state claim.

## Constitutional and objective core
- [ ] The constitutional kernel remains the supreme repo-local control regime.
- [ ] The workspace charter pair remains the repo-wide narrative + machine objective source.
- [ ] Mission is not the atomic execution primitive on supported consequential paths.
- [ ] Run contracts are the canonical consequential execution binding.
- [ ] Stage attempts and checkpoints are real runtime artifacts on ordinary supported runs.

## Authority
- [ ] `authority_engine` is a substantive implementation, not a re-export shell.
- [ ] ApprovalRequest / ApprovalGrant / ExceptionLease / Revocation / QuorumPolicy are runtime-consumed.
- [ ] DecisionArtifact and GrantBundle are emitted on ordinary authority decisions.
- [ ] Host adapters remain explicitly non-authoritative in both contract and behavior.
- [ ] Missing/expired/revoked authority fails closed.

## Durable lifecycle and evidence
- [ ] Every supported consequential run emits run-manifest, runtime-state, continuity, rollback posture, receipts, measurements, interventions, disclosure, and replay pointers.
- [ ] Resume/reset/compensation behavior is validated with recovery drills.
- [ ] External immutable replay index exists for the evidence classes that require it.
- [ ] Measurement and intervention records are mandatory, not optional references.

## Proof and lab
- [ ] Structural proof is green for the supported consequential envelope.
- [ ] Functional proof is green for the supported consequential envelope.
- [ ] Behavioral proof is green for the supported consequential envelope.
- [ ] Governance proof is green for the supported consequential envelope.
- [ ] Recovery proof is green for the supported consequential envelope.
- [ ] Maintainability proof is green for the supported consequential envelope.
- [ ] Lab scenarios, replay, shadow, fault, and adversarial packs are exercised and retained.

## Portability and support
- [ ] Support-target matrix is consumed by runtime routing.
- [ ] Adapter conformance is gating for host/model adapters.
- [ ] Browser/UI and broader API capability packs exist as governed pack surfaces.
- [ ] Unsupported tuples deny by default; stage_only tuples remain staged until proof lands.

## Disclosure and closeout
- [ ] RunCard exists for every supported consequential run.
- [ ] HarnessCard exists for the release claim and exactly matches the supported envelope.
- [ ] Retention and disclosure parity is proven.
- [ ] Governance overlays (`support-target-review`, `adapter-review`, `retirement-review`, `drift-review`, `closeout-reviews`) are release-blocking.
- [ ] Historical wave/migration artifacts are not carrying the burden of ordinary live proof.

## Simplification and adoption
- [ ] Transitional shims are removed or carry owner + review + retirement trigger.
- [ ] Non-load-bearing persona/roleplay surfaces are off the critical path.
- [ ] Brownfield retrofit/adoption playbook exists and has been walked through on a real target repo.
