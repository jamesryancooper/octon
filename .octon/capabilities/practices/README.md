# Capabilities Practices

## Purpose

Canonical practices surface for capabilities operating standards.

## Convention Authority

- This surface is the canonical home for capabilities naming and authoring conventions.
- Cross-domain baseline conventions come from `/.octon/conventions.md`.
- `_meta/architecture/` is reference architecture, not the canonical conventions surface.

## Contents

- `design-conventions.md` - Cross-cutting skill naming and authoring conventions.
- `services-conventions/` - Authoring and quality conventions for service capabilities.

## Boundary

Practice guidance belongs here. Runtime artifacts belong in `../runtime/`; governance contracts belong in `../governance/`.
