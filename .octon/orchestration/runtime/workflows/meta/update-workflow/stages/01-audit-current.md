# Step 1: Audit Current Workflow Surface

## Purpose

Establish the current canonical workflow state first, then inspect README drift.

## Actions

1. Resolve the canonical workflow unit from the provided path.
2. Load the current `workflow.yml`, `stages/`, and `README.md`.
3. Record current issues across contract completeness, stage quality, and README
   drift.

## Output

- Current-state audit of the workflow unit and README

## Proceed When

- [ ] Canonical workflow has been resolved
- [ ] README drift and contract gaps are documented
