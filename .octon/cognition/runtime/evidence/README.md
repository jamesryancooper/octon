# Runtime Evidence Map

Canonical runtime map of evidence bundle locations for decision, migration, and audit records.

## Purpose

- Make evidence discovery deterministic from cognition runtime surfaces.
- Link runtime records to generated evidence bundles under typed report roots in
  `/.octon/output/reports/`.
- Avoid ad-hoc evidence path resolution by humans and agents.

## Canonical Index

- `index.yml` - machine-readable evidence map.

## Upstream Record Sources

- Decisions record index: `/.octon/cognition/runtime/decisions/index.yml`
- Migration record index: `/.octon/cognition/runtime/migrations/index.yml`
- Audit record index: `/.octon/cognition/runtime/audits/index.yml`

## Generation

- Generated by: `/.octon/cognition/_ops/runtime/scripts/sync-runtime-artifacts.sh`
- Decision evidence entries are discovered from `/.octon/output/reports/decisions/<NNN>-<slug>/evidence.md` bundles.
- Migration and audit evidence entries are discovered from their runtime indexes and mapped to bundle evidence paths.
