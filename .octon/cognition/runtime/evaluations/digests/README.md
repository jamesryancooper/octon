# Runtime Evaluation Digests

Weekly scorecard digest artifacts for cognition runtime evaluation history.

## Purpose

- Keep periodic digest snapshots in one canonical runtime surface.
- Preserve week-over-week trend visibility without mutating scorecard contracts.
- Support deterministic audit trails for evaluation outcomes.

## Canonical Index

- `index.yml` - machine-readable discovery index for digest artifacts.

## Generation

- Generated by: `/.octon/cognition/_ops/runtime/scripts/sync-runtime-artifacts.sh`
- `index.yml` records are generated from digest markdown files in this directory.
