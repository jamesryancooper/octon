# Continuity Runs

Execution receipts, digests, and related run evidence.

## Purpose

`continuity/runs/` stores operational memory artifacts that support:

- auditability,
- governance evidence,
- incident and regression reconstruction.

This directory is append-oriented evidence storage, not a task state source of truth.
It exists to keep execution evidence deterministic enough to trust, observable enough to debug, and durable enough to support safe evolution.

## Canonical Policy

- Retention and class mapping: `retention.json`
- Architecture contract: `/.octon/continuity/_meta/architecture/runs-retention.md`
- Memory governance: `/.octon/agency/governance/MEMORY.md`

## What Belongs Here

- Per-run receipt bundles (`receipt*.json`, `digest*.md`, counters, ACP evidence)
- Deterministic run artifacts generated by runtime governance checks

Legacy note:

- Historical run bundles may still contain `acp-decision.json` artifacts.
- Those remain valid historical run-bundled evidence.
- New canonical orchestration decision records belong in `/.octon/continuity/decisions/`.

## What Does Not Belong Here

- Active task state (`tasks.json`)
- Handoff sequencing (`next.md`)
- General notes or ad hoc scratch artifacts

## Safety Rules

- Never store secrets or regulated data.
- Treat run artifacts as append-oriented evidence.
- Use retention policy classes for lifecycle handling; do not delete ad hoc.
