# Exploratory Inputs

`inputs/exploratory/**` stores non-authoritative material used to shape future
Octon work. These files may inform reviews, planning, synthesis, or proposal
promotion, but they never become runtime, policy, generated, state/control,
publication, retained evidence, or host-projection authority.

## Surfaces

| Surface | Role |
| --- | --- |
| `ideation/**` | Human-led exploratory notes and projects |
| `proposals/**` | Manifest-governed proposal packets |
| `plans/*.md` | Advisory implementation, migration, checklist, backlog, and assessment plans |
| `syntheses/*.md` | Research synthesis documents |
| `reports/<report-id>/` | Multi-file non-authoritative report sets |

Top-level `drafts/`, `packages/`, and root-level staged declarations are
retired. Use `syntheses/` for synthesis outputs and `reports/` for multi-file
report sets. Do not add new root-level exploratory files without a governed
taxonomy update.
