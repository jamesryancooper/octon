# Exploratory Reports

`inputs/exploratory/reports/<report-id>/` contains multi-file report sets that
do not match proposal, plan, synthesis, evidence, workflow, or extension-pack
contracts.

## Contract

- Each report directory must include `report.yml`.
- Use directory names that are either date-prefixed (`YYYY-MM-DD-<slug>/`) or
  stable topical slugs (`<slug>/`) when chronology is not the primary discovery
  key.
- Keep report contents self-contained and readable without depending on
  neighboring report directories.
- Do not use this surface for authoritative audit, migration, workflow,
  proposal, decision, evidence, or extension-pack bundles; those have dedicated
  roots.
- Reports are non-authoritative source material and must not be consumed as
  runtime, policy, generated, state/control, publication, retained evidence, or
  host-projection authority.

## Disposition

Reports may be converted into proposal packets, summarized into syntheses,
used as planning input, promoted into durable authored surfaces through a
separate governed Change, or retained as non-authoritative report history.
