# Octon Super-Root Semantics and Five-Class Taxonomy

This is a temporary, implementation-scoped architecture proposal for
`super-root-semantics-and-taxonomy`.
It translates the ratified super-root blueprint and Packet 1 design packet
into the repository's proposal format.
The proposal has been implemented in the repository and remains here as a
temporary non-canonical planning artifact until archival.
It is not a canonical runtime, documentation, policy, or contract authority.

## Purpose

- proposal kind: `architecture`
- promotion scope: `octon-internal`
- summary: Replace the current mixed-tree `.octon/` model with a class-first
  super-root that makes authored authority, raw inputs, mutable operational
  truth, and rebuildable generated outputs explicit and machine-enforceable.
