# Docs ARE Improvement Loop

Example extension-pack skill for proposal illustration only.

## When to Use

- The task is centered on improving repository documentation
- You want a reusable loop for analyze, refine, and evaluate work
- A docs-specific specialization should remain additive to core Octon

## Converted Source

This skill stands in for the removed ARE-related docs workflow assets from the
former scoped docs template. The phase mapping is summarized in
`references/phases.md`.

## Boundaries

- Additive only
- Does not introduce pack-local orchestration or governance authority
- Example is read-only; operational writes, if later added, must be rebased
  into Octon-owned destinations by the extension compiler

## Suggested Flow

1. Scope the documentation target.
2. Analyze current quality, coverage, and audit signals.
3. Refine the material.
4. Re-evaluate against the selected emphasis.
5. Escalate to adjacent template or tooling guidance only when needed.
