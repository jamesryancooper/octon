# ARE Phase Mapping

This reference condenses the removed ARE-related docs workflow set into one
extension skill shape.

## Preserved Concerns

- analyze a single document
- analyze existing audits
- refine the target content
- evaluate clarity and coverage
- stress-test the content set
- apply quality gates
- consult template guidance
- consult tooling guidance
- fold in best practices
- use quick-reference heuristics
- study worked examples
- reason about document sets, not only single files

## Extension-Pack Interpretation

- reusable docs specialization belongs in `skills/`
- docs starter assets belong in `templates/`
- none of this becomes extension-local governance or orchestration authority
