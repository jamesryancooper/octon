# Target-State Architecture

## Architectural thesis
Octon becomes a **constitutional, contract-governed control plane for autonomous work**.
The model is a pluggable reasoning component inside that system, not the system itself.

## Irreducible layers
1. Design Charter / Constitutional Layer
2. Intent / Objective Layer
3. Durable Control Layer
4. Policy / Authority Layer
5. Agency Layer
6. Runtime Layer
7. Verification / Evaluation Layer
8. Lab / Experimentation Layer
9. Governance / Safety Layer
10. Observability / Reporting Layer
11. Improvement / Evolution Layer

## Core target-state claims
- every consequential execution is bound to a run contract
- every material side effect is authority-routed before execution
- runtime is lifecycle-managed and resumable from artifacts
- all consequential runs emit proof, replay references, and disclosure artifacts
- support claims are explicit and bounded
- host surfaces are projections, not authority
- persona/roleplay surfaces are optional overlays unless they provide real boundary value
- obsolete compensating mechanisms are actively retired
