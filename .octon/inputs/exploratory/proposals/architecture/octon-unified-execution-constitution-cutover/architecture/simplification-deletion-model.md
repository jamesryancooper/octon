# Simplification, Deletion, and Evolution Model

## Keep stable
- class-root super-root
- fail-closed posture
- execution-authorization boundary
- route vocabulary including `STAGE_ONLY`
- mission as continuity/ownership layer
- state/control vs continuity vs evidence split
- structural and governance CI spine

## Keep replaceable
- model adapters
- host adapters
- capability pack implementations
- evaluator providers
- telemetry backends
- support-target defaults

## Load-bearing current agency surfaces
- single accountable default orchestrator
- routing and ownership rules
- delegation constraints
- memory discipline

## Transitional or removable surfaces
- persona-heavy identity layers in the kernel path
- `SOUL.md` as required ingress dependency
- host-native approval semantics
- mission-only execution assumption
- duplicated constitutional prose

## Build-to-delete rules
Every new compensating mechanism must carry:
- owner
- support scope
- value metric
- review date
- retirement trigger
- required ablation suite

## Retirement triggers
- model upgrade removes benefit
- support-target change invalidates workaround
- hidden-check / benchmark evidence shows no value
- intervention density does not worsen after removal
