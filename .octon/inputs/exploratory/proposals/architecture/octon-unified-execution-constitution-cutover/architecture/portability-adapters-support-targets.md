# Portability, Adapters, and Support Targets

## Portable kernel
Portable across repos and model families:
- class-root super-root and SSOT rules
- constitutional charter and precedence
- authority / runtime / disclosure schemas
- route semantics
- evidence classes
- proof-plane taxonomy
- support-target schema

## Non-portable adapters
Allowed and expected to vary:
- model adapters
- host adapters
- capability packs
- evaluator providers
- browser drivers / API connectors
- telemetry backends
- repo-local overlays

## Host adapters
Canonical hosts to support:
- GitHub
- CI
- local CLI
- Studio

## Model adapters
Every model family must ship:
- adapter manifest
- conformance suite refs
- contamination / reset policy
- support-tier declarations
- known limitations

## Capability packs
Governed packs by surface:
- repo
- git
- shell
- browser
- api
- telemetry

## Support-target matrix axes
- model tier
- workload tier
- language/resource tier
- locale tier

## Admission rules
New support tiers require:
1. adapter contract
2. conformance pass
3. proof-plane minimums
4. disclosure update
5. governance approval
