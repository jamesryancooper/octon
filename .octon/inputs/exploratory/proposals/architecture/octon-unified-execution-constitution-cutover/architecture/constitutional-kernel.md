# Constitutional Kernel

## Purpose
Create one supreme repo-local constitutional kernel that unifies what is currently distributed
across root manifests, architecture prose, bootstrap guidance, runtime policy interfaces, and
workflow glue.

## Proposed canonical root
`/.octon/framework/constitution/**`

## Required artifacts
- `CHARTER.md`
- `charter.yml`
- `precedence/normative.yml`
- `precedence/epistemic.yml`
- `obligations/fail-closed.yml`
- `obligations/evidence.yml`
- `ownership/roles.yml`
- `contracts/**`
- `support-targets.schema.json`

## Constitutional non-negotiables
1. No material execution without a bound run contract.
2. No material side effect before authority routing and grant issuance.
3. Host adapters may not become a second control plane.
4. Raw `inputs/**` may not become direct runtime or policy dependencies.
5. `generated/**` remains derived-only.
6. Every consequential run emits required evidence and disclosure.
7. Unsupported support tiers fail closed.
8. Hidden human repair is prohibited.
9. Every compensating mechanism must carry retirement metadata.

## Dual precedence
### Normative authority
1. external/non-waivable obligations and break-glass revocations
2. constitutional charter and fail-closed obligations
3. support-target matrix and repo governance policy families
4. approvals / grants / leases / revocations for the current run
5. workspace charter
6. mission charter
7. run contract
8. capability / model / host adapter contracts
9. authored repo architecture/spec artifacts
10. informative docs, generated summaries, chat residue, model priors

### Epistemic grounding
1. live runtime observations and validated receipts
2. deterministic validators and measurements
3. current repo/worktree state
4. mutable `state/control/**` truth
5. receipt-valid `generated/effective/**`
6. provenance-backed continuity artifacts
7. authored documentation and charters
8. conversation/operator commentary
9. model priors
