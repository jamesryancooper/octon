# Risks, Tradeoffs, and Open Questions

## Risks
- constitutional duplication during migration
- temporary complexity while old and new authority paths dual-write
- external replay storage increases operational obligations
- support-target honesty may feel narrower before adapters mature
- deletion of familiar persona surfaces may create ergonomic resistance

## Tradeoffs
- preserving strong current structure vs accelerating simplification
- Git reviewability vs replay completeness
- explicit support boundaries vs optimistic universality
- run-first normalization vs mission-centric continuity habits

## Open questions to resolve during implementation
- exact external immutable replay backend and retention windows
- exact threshold for mission-required vs run-only autonomy classes
- whether `runtime-state` remains the canonical run manifest equivalent or is split
- how much legacy cognition material remains as information-only history after constitutional extraction
