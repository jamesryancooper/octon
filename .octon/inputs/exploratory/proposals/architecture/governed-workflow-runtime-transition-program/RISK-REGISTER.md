# Risk Register

_Status: Draft parent-program risk register_

| Risk | Severity | Mitigation |
| --- | ---: | --- |
| Parent program becomes a second conceptual control plane | High | Parent coordinates only; child manifests, receipts, validation verdicts, promotion targets, and archive metadata remain child-owned. |
| Program creates proposal sprawl | Medium | Keep only eight required children plus three deferred/lab-only evaluations; require mutation evidence to add more. |
| Framing packet is promoted before prerequisites are proven | High | Gate canonical runtime-terminology cutover behind `migration-cutover-compatibility-retirement`. |
| Governed Workflow Runtime becomes terminology churn | Medium | Require terminology guardrail child and compatibility-retirement evidence. |
| Workflow statechart work creates a new state root | High | Bind statecharts to existing `state/control/**` and Run Lifecycle v1 until validated cutover. |
| Task-specific execution harness duplicates run contracts | Medium | Define harness as compiled envelope, not independent authority. |
| Agent-node contract remains too vague | High | Require schemas, model-call receipts, output validation, budgets, tool allowlists, and forbidden authority claims. |
| Replay/rollback claims overstate runtime capability | High | Require workflow history, replay validator, idempotency, retry, and compensation receipts before claims widen. |
| Effect-token enforcement coverage misses material paths | High | Require PEP inventory, negative bypass tests, and effect-token coverage receipts. |
| Connector/tool/MCP availability is treated as permission | High | Connector admission child must preserve availability-is-not-permission rule. |
| Durable Objects become hidden authority | High | Keep Durable Object work deferred/lab-only and rejected as authority/control/evidence. |
| External workflow engines become rival runtime authority | High | Keep external workflow engines deferred/lab-only and adapter-only. |
| Generated diagrams or read models become control truth | Medium | Generated projection validation must mark diagrams/read models derived-only. |
| Conversation artifacts are mistaken for proof | Medium | Treat determinism conversations as non-authoritative source lineage only. |
| Validators exist without runtime enforcement | High | Require implementation-conformance and drift/churn receipts for implemented child packets. |
| Lifecycle Autopilot route execution is unavailable in current repo state | Medium | Record the blocker, create packets from canonical lifecycle/scaffolding conventions, and rely on proposal validators until the effective extension catalog issue is fixed. |
| Child packets are structurally valid but not implementation-ready | Medium | Keep child implementation-grade receipts failing until child-specific contracts, schemas, validators, and promotion evidence exist. |
