# Target Architecture

_Status: Draft parent-program target_

This parent program targets a safely claimable Governed Workflow Runtime
architecture only after child packets prove the required contracts, validators,
receipts, and cutover evidence.

## Program-Level Target State

1. Octon remains a Constitutional Engineering Harness.
2. Governed Workflow Runtime becomes claimable only after child packet evidence
   proves workflow-first runtime contracts.
3. Existing Run Lifecycle v1 remains canonical until statechart parity and
   cutover evidence exist.
4. Task-specific execution harnesses are defined by schema and compilation
   receipts before being claimed as live.
5. Agent nodes are defined as bounded, evidenced activity nodes before being
   used as canonical runtime language.
6. Workflow history, replay, idempotency, retry, and compensation semantics are
   explicit and validator-covered before durable replay/rollback claims widen.
7. Connector/tool/MCP availability remains non-permission.
8. Durable Objects and external workflow engines remain rejected as authority
   and deferred or lab-only as adapters.
9. Compatibility retirement happens only after accepted migration/cutover proof.

## Non-Target State

The parent program does not target an external workflow engine, Durable Object
control plane, MCP authority model, agent-owned scheduler, generated projection
control plane, or universal transactionality claim.
