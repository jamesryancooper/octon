# Octon-Wide Delegated Governance Architecture

This proposal defines an Octon-wide target stance for delegated governance.

The stance is proof-first: automation should execute by default when authority
is proven through contract, evidence, scope, authority zone, replay safety,
retained receipts, and fail-closed validation. Human approval should remain
only for typed exception boundaries that cannot be machine-proven.

This packet is architecture-only. It does not change runtime behavior, schemas,
authority contracts, connector permissions, workflows, state/control truth,
retained evidence, or generated projections.
