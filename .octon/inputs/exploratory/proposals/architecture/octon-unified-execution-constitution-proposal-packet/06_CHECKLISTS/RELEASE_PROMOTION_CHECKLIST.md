# Release Promotion Checklist

A release may not promote if any item below is incomplete.

## Control and authority
- [ ] Constitutional kernel unchanged except by explicit approved amendment.
- [ ] Support-target review passed.
- [ ] Adapter review passed.
- [ ] Retirement review passed.
- [ ] Drift review passed.
- [ ] Closeout reviews passed.

## Runtime and evidence
- [ ] Ordinary supported consequential runs succeeded on the live path.
- [ ] Run evidence roots contain receipts, measurements, interventions, replay pointers, and disclosure.
- [ ] External replay index / evidence lifecycle obligations satisfied.

## Proof and lab
- [ ] Required proof-plane workflows/suites are all green.
- [ ] Required lab scenario/shadow/fault evidence exists for the supported claim.
- [ ] Hidden/held-out evaluator sets were not contaminated by harness-tuning changes.

## Disclosure
- [ ] RunCard set is complete for representative supported runs.
- [ ] HarnessCard matches the exact supported tuple envelope.
- [ ] Known limits section is current and honest.
- [ ] No unsupported or stage_only tuple is represented as supported.

## Simplification
- [ ] New transitional mechanisms have owners and retirement triggers.
- [ ] No retired or non-live artifact is still on the release critical path.
