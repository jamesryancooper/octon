# Acceptance Criteria

- Parent program is accepted as coordination-only and does not own child
  implementation truth.
- Child registry names every required migration domain and keeps children as
  sibling packets.
- Each child declares write scope, evidence gates, rollback posture, validation
  plan, and proposal-local non-authority stance.
- Parent validators pass for proposal standard, architecture, program
  structure, child readiness, and implementation readiness.
- Closeout remains blocked until all required children reach terminal outcomes
  with fresh child-owned receipts.
- Generated outputs and read models remain observational and cannot become
  control truth.
