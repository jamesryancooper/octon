# Target Architecture

The target architecture is an Octon-wide governance model where routine
execution is machine-delegated when retained proof establishes authority through
contract, invocation, evidence, scope, authority zone, replay or compensation
safety, required receipts, and fail-closed validation.

Human approval remains only as a typed exception grant for a boundary that
cannot be machine-proven: scope expansion, policy override, unresolved risk
acceptance, governance mutation, contradictory evidence resolution, stale
evidence acceptance, authority ambiguity, unsafe resume, or external
irreversible effect.

The lifecycle clean-break migration is the reference implementation. The child
packets generalize the same proof-first stance without forcing every non-
lifecycle surface to reuse the lifecycle route schema verbatim.

Generated outputs, read models, proposal-local receipts, dashboards, tool
availability, chat history, and agent output may support evidence, but they
never grant authority.
