# Fully Unified Execution Constitution for Governed Autonomous Work

This is a temporary, implementation-scoped architecture proposal for
`fully-unified-execution-constitution-for-governed-autonomous-work`. It does
not become canonical runtime, governance, policy, or documentation authority
until its outputs are promoted into durable `/.octon/**` surfaces.

## Purpose

- proposal kind: `architecture`
- promotion scope: `octon-internal`
- summary: Turn Octon from a distributed repository constitution into a fully
  unified execution constitution where every consequential autonomous action is
  explicitly scoped, authorized, executed, verified, replayable, and disclosed.
- current status: `accepted`

## What This Proposal Preserves

- the five-class super-root:
  `framework / instance / inputs / state / generated`
- authored authority only in `framework/**` and `instance/**`
- raw `inputs/**` as non-authoritative and excluded from runtime and policy
  resolution
- `generated/**` as derived-only even when runtime consumes fresh, receipt-
  backed effective outputs
- fail-closed execution posture
- mission as the continuity, ownership, and long-horizon autonomy container
  even when run contracts become the atomic execution unit
- adapter projections as non-authoritative host glue rather than control-plane
  authority

## Exit Path

Promote one coherent constitutional kernel, run-contract model, authority
router, runtime evidence model, assurance plane expansion, lab surfaces, and
disclosure model into durable `/.octon/**` paths. Once those durable surfaces
stand on their own and no longer depend on proposal-local guidance, archive
this proposal and retain the current `resources/*.md` files as historical
research lineage only.
