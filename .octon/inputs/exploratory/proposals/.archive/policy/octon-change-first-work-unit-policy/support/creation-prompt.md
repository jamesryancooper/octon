# Creation Prompt

Create a concise Octon proposal packet for the Change-first default work unit stance.

The packet should codify that Octon targets solo developers, so Pull Requests are optional publishing and review outputs rather than the default work unit. It should define Change as the default work unit, specify practical routing for direct-main, branch-without-PR, and branch-with-PR paths, and identify the policy, workflow, adapter, manifest, and validation surfaces that need alignment.

The packet must be non-authoritative proposal material under the exploratory proposal tree and must identify the intended durable policy location and downstream references for promotion.
