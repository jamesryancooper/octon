# Prompt Delta — Original to Revised

## Why the prompt was revised

The original prompt was strong on ambition but materially wrong on three points:

1. **It presumed final completion certification.** The audit explicitly says Octon is not yet target-state complete, so the prompt could not responsibly demand evidence-backed certification that target-state attainment has already occurred.
2. **It required a single atomic big-bang migration.** The packet and audit both imply staged rollout, transition handling, and support-target boundedness. A forced clean-break migration would conflict with repository reality and truth-preserving governance.
3. **It implicitly treated some now-substantive repo assets as if they were still greenfield design territory.** The revised prompt preserves and hardens existing constitutional, runtime, adapter, lab, observability, and disclosure surfaces.

## Major changes

### 1. Present completion claim → future evidence-gated certification
**Original posture:** demanded “complete, evidence-backed certification that the target state has been achieved with no unresolved gaps.”

**Revised posture:** requires defining the conditions for **future** closure certification and forbids overclaiming present completion unless the supplied evidence genuinely supports it.

### 2. Big-bang migration → phased truth-preserving rollout
**Original posture:** single, atomic, big-bang, clean-break migration with no intermediate partially compliant states.

**Revised posture:** phased migration with:
- explicit transition modes,
- compatibility shims where bounded and temporary,
- promotion/restriction/de-admission rules,
- exclusion of transitional surfaces from the live completion claim.

### 3. Greenfield redesign → preserve-and-harden
**Original posture:** susceptible to greenfield redesign energy.

**Revised posture:** explicitly preserves the current substantive repo assets and requires only hardening, normalization, re-bounding, simplification, or deletion where justified.

### 4. Weak audit claims corrected by stronger packet refinements
The revised prompt instructs the model to incorporate the packet’s narrower refinements where they materially correct weaker audit claims, especially around:
- already-live exception-lease state,
- already-partial attempt/stage materialization,
- already-existing build-to-delete validators,
- closure artifacts that exist but must be demoted from truth-source to generated summary.

### 5. Completion truth explicitly tied to evidence graph
The revised prompt adds claim-truth requirements around:
- admissions,
- proof bundles,
- adapter conformance,
- replay evidence,
- retirement state,
- generated closure outputs.

### 6. Archive requirement made concrete
The revised prompt specifies a minimum archive structure so the output is a real handoff artifact rather than only a long-form essay.

## Net effect

The revised prompt is harder to satisfy dishonestly, more faithful to the audit and packet, and better aligned with an implementation-grade architecture program for Octon completion.
