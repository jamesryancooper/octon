# Risk Register

| Risk | Harm | Mitigation |
| --- | --- | --- |
| Second orchestrator | Splits accountability and weakens the orchestrator role | Define the planner as a tool of the single orchestrator with no execution authority. |
| Plan as authority | Violates class-root and mission authority rules | Validators reject plan refs as runtime authority and require mission digest binding. |
| Plan bypasses authorization | Breaks the engine-owned execution gate | Leaves compile only to candidates; material effects still require `authorize_execution`. |
| Proposal packet becomes runtime dependency | Violates proposal non-canonical rule | Promotion must translate accepted content into durable targets and remove active proposal-path dependencies. |
| Generated plan treated as control truth | Creates a derived second control plane | Generated planning projections carry non-authority metadata and forbidden-consumer posture. |
| Planning bloat | Creates fake progress and operator load | Enforce depth, breadth, rolling-wave, and non-useful revision limits. |
| Infinite decomposition | Avoids execution and evidence | Executability and readiness tests force execute, block, defer, or escalate after bounded attempts. |
| Duplicate work | Creates competing branches for the same output | Validate duplicates by scope, output, dependency, and action class. |
| Stale plan state | Diverges from mission or run evidence | Mission digest and evidence drift block compile. |
| Support widening | Expands live claims without proof | Plan nodes can cite support refs but cannot admit capabilities or widen support targets. |
| Weak assumptions | Encodes guesses as facts | Unresolved assumptions compile only to discovery work or remain blocked. |
| Rollback confusion | Plan appears to replace run rollback truth | Rollback truth stays in run control and evidence; plan status is derived linkage. |
