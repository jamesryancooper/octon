# Cutover Checklist

## Before cutover

- [ ] Proposal packet validated.
- [ ] Implementation run contract created.
- [ ] Promotion targets reviewed for proposal-path backreferences.
- [ ] Baseline architecture health captured.
- [ ] Current generated/effective publication state captured.
- [ ] Current support target/proof state captured.
- [ ] Current extension active/quarantine state captured.

## During cutover

- [ ] Add runtime-resolution spec/schema/instance selector.
- [ ] Generate runtime-effective route bundle and lock.
- [ ] Normalize support admission/dossier paths.
- [ ] Generate support matrix and support cards.
- [ ] Compile pack routes.
- [ ] Compact extension active state.
- [ ] Regenerate extension catalog/artifact map/generation lock.
- [ ] Update runtime resolver and authorization boundary.
- [ ] Add validators and negative-control tests.

## After cutover

- [ ] Run all validation groups in `architecture/validation-plan.md`.
- [ ] Confirm generated/effective outputs have current publication receipts.
- [ ] Confirm no raw input, proposal, generated/cognition, or host projection path participates in runtime authority.
- [ ] Confirm support stage-only/unadmitted surfaces are not live support.
- [ ] Confirm shims are registered with retirement triggers.
- [ ] Generate closure certification.
- [ ] Archive proposal only after durable targets stand alone.
