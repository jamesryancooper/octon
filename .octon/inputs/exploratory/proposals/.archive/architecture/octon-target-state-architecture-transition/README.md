# Octon Target-State Architecture Transition

This is a manifest-governed Octon architecture proposal packet for moving the live repository architecture from its current strong-but-incomplete state to the target-state governed runtime architecture.

The packet is implementation-oriented and promotion-safe. It is not canonical authority. It may guide promotion into durable `.octon/**` surfaces, but runtime, policy, publication, support, and closure authority remain outside this proposal workspace.

## Controlling thesis

Octon does **not** need re-foundation. It needs focused gap-closing plus moderate runtime restructuring:

1. preserve the constitutional super-root, class-root model, mission/run split, generated/authored discipline, and support-target tuple model;
2. prove complete `authorize_execution` coverage over material side effects;
3. split runtime command and authorization internals into auditable modules/phases;
4. harden proof-plane sufficiency and evidence completeness;
5. generate human/agent navigation maps from the structural registry;
6. retire transitional compatibility projections on an owner/consumer/expiry schedule.
