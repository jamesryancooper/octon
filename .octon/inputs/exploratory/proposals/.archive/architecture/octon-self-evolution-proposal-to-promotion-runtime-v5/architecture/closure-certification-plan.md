# Closure Certification Plan

Closure certification requires:

1. all artifacts exist in the correct roots;
2. no proposal-path dependency remains in durable promoted outputs;
3. candidate/proposal/promotion/recertification lineage is retained;
4. two consecutive validation passes produce no blocking issues;
5. negative controls prove no self-authorization;
6. final recertification status is retained under `state/evidence/evolution/recertifications/**`;
7. the Evolution Ledger indexes closure without replacing source evidence.
