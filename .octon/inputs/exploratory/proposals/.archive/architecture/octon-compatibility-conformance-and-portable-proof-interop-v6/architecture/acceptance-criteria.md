# Acceptance Criteria

- [x] Octon Compatibility Profile is defined and validated.
- [x] External project compatibility inspection is implemented with explicit contracts.
- [x] Safe external project adoption is specified with no-blind-copy behavior.
- [x] Portable Proof Bundle is defined and validated.
- [x] Attestation Envelope is defined and validated.
- [x] Proof Acceptance Record is defined and validated.
- [x] Proof import/export lifecycle is specified and wired.
- [x] Attestation verify/accept/reject lifecycle is specified and wired.
- [x] Imported proof is retained as evidence only.
- [x] Attestation acceptance cannot mint authority.
- [x] Trust-domain hook exists and fails closed.
- [x] Revocation and expiry semantics exist.
- [x] Runtime/CLI additions are specified and implemented.
- [x] Negative tests prevent imported proof or generated trust views from bypassing local authority.
- [x] Deferred federation compact, delegation, certification, and multi-org trust scope is explicit.
