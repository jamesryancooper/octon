# Execution Constitution Conformance Card

This proposal does not create execution authority. It creates compatibility classification and proof exchange surfaces.

All material execution remains under the run lifecycle and `execution-authorization-v1` boundary. Imported proof, attestations, certifications, and federation artifacts may become retained evidence only after local verification and acceptance. They do not authorize execution.

Generated trust dashboards, maps, and status views are derived-only and forbidden as policy/runtime authority.
