# Governance and Safety Gates

## Human-approved

- initial adoption writes into an unadopted repo;
- workspace charter creation or material rewrite;
- support-target widening;
- first live use of any unadmitted connector/tool/MCP;
- destructive or irreversible actions;
- protected-zone mutation;
- credential or external service enablement;
- policy ambiguity resolution;
- risk acceptance;
- mission scope expansion or lease extension;
- closeout with unresolved risk.

## Runtime-authorized

- repo mutation;
- control/evidence mutation outside the preflight lane;
- executor launch;
- service/tool/MCP invocation;
- shell execution;
- generated-effective publication;
- protected CI actions;
- outbound HTTP;
- model-backed execution;
- capability-pack activation.

## Evidenced

- adoption/classification;
- orientation;
- Project Profile source facts;
- per-engagement objective candidate and charter reconciliation;
- Work Package compilation;
- support/capability posture;
- context-pack request;
- Decision Requests;
- run-contract candidate;
- authorization decisions;
- run execution and closeout.

## Fail-closed conditions

- missing authority binding;
- missing or stale context evidence;
- invalid support tuple;
- unadmitted capability pack;
- unsupported connector;
- missing rollback posture;
- missing approval;
- missing per-engagement objective or charter binding;
- Objective Brief attempts to rewrite workspace-charter authority;
- generated/effective handle direct read;
- proposal-local or `inputs/**` runtime dependency.
