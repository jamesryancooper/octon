# Risk Register

| Risk | Impact | Mitigation |
|---|---|---|
| Compiler becomes rival control plane | Undermines run lifecycle and authorization boundary. | Work Package is candidate/readiness artifact only; material execution stays with run contracts and authorization. |
| Project Profile becomes unreviewed authority | Repo facts could be wrong and durable. | Raw orientation in evidence; durable profile promoted only through reconciliation and explicit status. |
| Decision Request bypasses approval roots | Approval semantics fragment. | Decision Request must resolve into canonical approvals/exceptions/revocations/risk-acceptance records. |
| Preflight lane mutates repo | Unsafe adoption behavior. | Lane contract forbids project-code mutation and external effects. |
| Connector posture overclaims MCP support | Unsafe external tool use. | v1 stage-only, blocked, or denied for non-admitted connectors. |
| Engagement Objective Brief becomes workspace authority | Objective shaping could bypass charter governance. | Objective Brief remains per-engagement candidate/control state and cannot rewrite the workspace-charter pair. |
| Evidence Profile weakens closeout | Consequential runs under-evidenced. | Evidence Profile cannot reduce required run lifecycle/evidence-store obligations for repo-consequential work. |
| Generated projection consumed as authority | Hidden control-plane drift. | Validators reject generated-as-authority consumption. |
| Operator UX hides critical stop conditions | Safety becomes invisible. | Status surfaces must show blocked/staged/denied reason codes and Decision Requests. |
