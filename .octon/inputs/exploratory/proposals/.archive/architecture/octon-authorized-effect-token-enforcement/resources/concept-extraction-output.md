# Concept Extraction Output

## Extracted concept

**Authorized Effect Token enforcement across every material path family**.

## Extracted operating principle

Authorization must become a consumable typed runtime object, not merely a prior decision artifact. The model, Agent, adapter, shell wrapper, generated output, or host surface must not be able to cause a material effect by asserting that a grant exists. The callee must require and verify the specific effect token.

## Extracted implementation primitives

- typed `AuthorizedEffect<T>` transport artifact;
- authority-owned minting from successful grants;
- canonical token records under Run control state;
- token lifecycle journal events/items;
- token consumption receipts;
- non-serializable `VerifiedEffect<T>` guard;
- side-effect API signature hardening;
- material side-effect inventory;
- negative bypass tests;
- support-target token proof.

## Extracted effect families

- repo mutation;
- generated-effective publication;
- state/control mutation;
- evidence mutation;
- executor launch;
- service invocation;
- protected CI check;
- extension activation;
- capability-pack activation;
- any declared material path not covered by the v1 classes must be mapped or added as a v2 class.
