# Non-Goals and Exclusions

This packet intentionally excludes:

- new model adapters
- browser automation
- external API autonomy expansion
- irreversible external action support
- product UI redesign
- generic observability dashboards
- broad memory architecture
- multi-agent orchestration redesign
- marketing copy or homepage rewrite
- making proposal packets authoritative
- promoting archived proposal ideas as current capability
- changing root ingress adapters
- changing `AGENTS.md`, `CLAUDE.md`, or `.octon/instance/ingress/AGENTS.md`
- changing `.github/workflows/**` under this octon-internal proposal unless a
  later separate repo-scope proposal admits that change

## Out of scope after migration

Even after this migration, Octon should not claim universal autonomy. It will
have a stronger runtime governance foundation for admitted surfaces, not a broad
license to treat every future-facing design as live support.

## Explicitly preserved

- support-limited autonomy
- generated non-authority
- input/proposal non-authority
- state/control, state/evidence, and state/continuity separation
- engine-owned authorization
- proof-backed live support claims
