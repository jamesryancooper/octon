## Materially real today

- Octon identifies itself as a Constitutional Engineering Harness with a
  Governed Agent Runtime.
- The charter defines a bounded admitted finite support universe and requires
  surfaces outside live conditions to be stage-only, unadmitted, reduced, or
  unsupported.
- The architecture spec separates authored authority, mutable control, retained
  evidence, continuity, generated/effective handles, generated read models,
  compatibility projections, proposal inputs, and raw inputs.
- Runtime specs define engine-owned authorization, typed effect tokens, boundary
  coverage, run lifecycle, evidence completeness, and operator read-model rules.
- Assurance scripts and tests already exist for many related behaviors.

## Partially implemented or emerging

- Support truth is distributed across declarations, proofs, generated route
  bundles, pack routes, matrices, cards, and disclosures.
- Typed effect-token contracts exist, but visible runtime/code materialization is
  incomplete relative to the spec.
- Operator read-model principles exist, but a per-run health model is not yet a
  complete generated surface.

## Target-state or aspirational

- Archived proposal packets and exploratory inputs are not implementation proof.
- Broad future-facing adapters, browser autonomy, external API autonomy, and
  frontier/studio surfaces are not in scope unless active support and route
  artifacts admit them.
