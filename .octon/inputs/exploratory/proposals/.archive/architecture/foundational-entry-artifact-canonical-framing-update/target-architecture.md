# Target Architecture

_Status: In-review proposal packet artifact_


## Target entry-artifact state

After this packet is implemented, foundational entry artifacts should introduce Octon in this order:

1. Octon is a Constitutional Engineering Harness.
2. Its core runtime is best understood as a Governed Workflow Runtime.
3. Workflow state owns control flow.
4. Each admitted workflow receives a task-specific execution harness.
5. Agents may participate only as bounded, evidenced activity nodes.
6. Engine-owned authorization and typed effect tokens control material effects.
7. Deterministic context packs and retained evidence make work inspectable and replayable.
8. Generated projections and raw inputs remain non-authoritative.
9. Connectors/tools are admitted operations, not ambient permissions.
10. Durable Objects, if ever introduced, are live coordination adapters only.
