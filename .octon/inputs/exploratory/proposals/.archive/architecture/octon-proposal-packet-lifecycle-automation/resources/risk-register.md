# Risk Register

| Risk | Severity | Mitigation |
| --- | --- | --- |
| Automation becomes a rival proposal authority | High | Shared contracts must state that proposal manifests, standards, validators, and durable targets outrank prompts and generated artifacts. |
| Whole-universe scope becomes too broad to implement coherently | High | Keep one extension pack and route-specific bundles; require atomic publication and complete validation before merge. |
| Closeout route performs unsafe GitHub or branch operations | High | Require explicit checks for green CI, resolved review conversations, intended staging, clean worktree, and synced origin. |
| Generated prompts hard-code stale Octon framing | Medium | Meta-prompts derive framing from live repo and packet inputs; only packet-specific generated prompts may include current framing. |
| Registry regeneration accidentally includes unrelated untracked packets | Medium | Cutover checklist requires review of visible proposal packets before registry write. |
| Validation fixtures under-cover manual scenarios | Medium | Bundle matrix must map every manual prompt class to at least one scenario. |
| Host projections drift from generated effective capability routes | Medium | Run capability and host projection publication validators in closeout. |
| Correction loop hides unresolved findings as deferrals | Medium | Deferrals require explicit rationale, owner, and follow-up route. |
