# Rejection Ledger

This hard-cut proposal rejects the following.

## Ontology rejections

- `agent` as canonical Octon noun
- `assistant` as canonical Octon noun
- `team` as canonical Octon noun
- `actor` as canonical Octon noun
- durable `subagent`
- persona authority
- identity-file authority
- SOUL-style runtime authority

## Architecture rejections

- dual agency/execution-role paths
- compatibility bridges
- mission-only execution
- generated cognition as memory or authority
- host UI, labels, comments, checks, or chat as authority
- workflow-as-thinking-recipe
- token-era routing as governance
- multi-agent swarm framing
- default planner/generator/evaluator roleplay topology

## Support/proof rejections

- unsupported live browser/API claims
- live support without tuple admission
- support claims without dossiers
- RunCards/HarnessCards written from prose
- browser/API actions without replay, redaction, egress lease, and compensation posture
- experimental external adapters in active discovery

## Control-plane rejections

- rival control plane outside `.octon/**`
- MCP as replacement for Octon runtime lifecycle
- runtime event bus as authority instead of projection
