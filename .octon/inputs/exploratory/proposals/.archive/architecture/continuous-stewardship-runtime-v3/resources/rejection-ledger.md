# Rejection Ledger

## Rejected: Infinite Agent Loop

Rejected because it violates controlled autonomy, closure, evidence, and
fail-closed posture.

## Rejected: Campaign as Stewardship

Rejected because campaigns are optional coordination rollups and must not become
a second mission system or stewardship model.

## Rejected: Stewardship Direct Execution

Rejected because all material execution must remain governed by v2 Mission
Runner and run authorization.

## Rejected: Generated Stewardship Memory as Authority

Rejected because generated outputs are derived projections only.
