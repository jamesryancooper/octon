# Full Concept Integration Assessment

## Adopt

- Stewardship Program
- Stewardship Epoch
- Stewardship Trigger
- Stewardship Admission Decision
- Idle Decision
- Renewal Decision
- Stewardship Ledger
- Stewardship Evidence Profile

These concepts are not currently live as usable Octon mechanisms and are needed
for long-running governed availability.

## Adapt

- Stewardship-Aware Decision Request: extends v1/v2 Decision Request semantics to
  program, epoch, trigger, renewal, and stewardship closeout gates.
- Campaign Coordination Hook: adapts existing campaign criteria into a gate that
  may emit a campaign candidate, but preserves campaigns as optional and
  deferred by default.

## Defer

- Broad external event ingestion.
- Multiple stewardship programs per repo.
- Multi-repo stewardship federation.
- Automatic campaign promotion.
- Fully unattended unconstrained long-horizon work.

## Reject

- Infinite agent loop.
- Direct material execution from stewardship.
- Campaign as stewardship replacement.
- Generated stewardship summary as authority.
