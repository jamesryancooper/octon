# MVP Boundary

The v3 MVP includes:

- one active Stewardship Program per repo/workspace;
- one active Stewardship Epoch per program;
- scheduled-review and human-objective triggers;
- optional repo-change or validation-age-out triggers only if already supported;
- Stewardship Admission Decisions;
- Idle Decisions;
- Renewal Decisions;
- Stewardship Ledger;
- Stewardship Evidence Profile;
- stewardship-aware Decision Requests;
- handoff to v2 Mission Runner;
- no direct material execution;
- campaigns remain deferred unless promotion criteria are met.

The MVP excludes multiple stewardship programs, broad external events,
effectful MCP/API/browser autonomy, deployment automation, credential
provisioning, multi-repo stewardship, autonomous governance changes, silent
campaign promotion, destructive operations, and infinite loops.
