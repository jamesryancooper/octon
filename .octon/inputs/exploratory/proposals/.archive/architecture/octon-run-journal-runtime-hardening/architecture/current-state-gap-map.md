# Current-State Gap Map

This map is focused only on the Run Journal hardening step.

| Current surface | Current strength | Gap | Blocking effect | Required change |
|---|---|---|---|---|
| `.octon/framework/constitution/contracts/runtime/run-event-ledger-v1.schema.json` | Establishes ledger manifest fields and canonical run ledger family. | Manifest lacks sufficient integrity semantics such as sequence boundary, previous/current hash chain, canonical side-artifact refs, and redaction lineage. | Cannot prove tamper-evidence, causal completeness, or safe evidence compaction. | Add `run-event-ledger-v2` with hash-chain, event count, schema refs, first/last refs, redaction refs, closeout snapshot refs, and drift status. |
| `.octon/framework/constitution/contracts/runtime/run-event-v1.schema.json` | Defines canonical event shape and several lifecycle/governance events. | Payload is too generic; event ordering, actor, causal parent, command ref, state transition, and side-effect refs are not strong enough. | Replay, policy replay, and support-target admission remain weak. | Add `run-event-v2` with typed event envelope and required causal/integrity refs. |
| `.octon/framework/constitution/contracts/runtime/runtime-state-v1.schema.json` | Defines mutable runtime state. | Status enum differs from engine run-lifecycle terms and does not fully encode derivation from journal. | Drift between lifecycle specs, runtime state, and engine behavior remains possible. | Add `runtime-state-v2` as explicitly derived view with `source_ledger_ref`, `last_applied_event_id`, `last_applied_sequence`, and drift status. |
| `.octon/framework/constitution/contracts/runtime/state-reconstruction-v1.md` | Correctly states ledger wins over runtime-state conflicts. | Needs operational validator and runtime crate alignment. | Principle exists but may not be enforced. | Add v2 reconstruction contract and validator script. |
| `.octon/framework/engine/runtime/spec/runtime-event-v1.schema.json` | Runtime event schema exists. | Uses dot-named events and thinner required fields than constitutional run events. | Runtime telemetry can diverge from constitutional ledger. | Align runtime events to canonical run events or define legacy aliases with mandatory normalization. |
| `.octon/framework/engine/runtime/spec/run-lifecycle-v1.md` | Defines run roots and fail-closed lifecycle states. | State terms differ from constitutional runtime-state; lifecycle transitions not tied to event IDs and hash chain. | Resume/replay/checkpoint correctness remains under-specified. | Add transition table linked to canonical events and `runtime-state-v2`. |
| `.octon/framework/engine/runtime/spec/evidence-store-v1.md` | Defines closeout evidence obligations. | Needs exact snapshot relationship between live control journal and retained evidence. | Evidence bundle may be complete-looking without matching control truth. | Require evidence snapshot hash match and closeout event. |
| `.octon/framework/engine/runtime/spec/operator-read-models-v1.md` | Preserves generated non-authority. | Needs stronger requirement that operator views derive from journal/evidence only. | Operator summaries could become de facto runtime input. | Add source-ref/freshness/journal-hash requirements. |
| `.octon/framework/engine/runtime/crates/runtime_bus` | Existing runtime service family suggests bus role. | Needs single append path and schema/sequence/hash enforcement. | Multiple writers could bypass canonical journal. | Make runtime bus the only journal append service. |
| `.octon/framework/engine/runtime/crates/replay_store` | Existing runtime service family suggests replay role. | Needs journal-first reconstruction and dry-run replay safety. | Replay can be incomplete or side-effect unsafe. | Implement journal reconstruction and side-effect-denied replay default. |
| `.octon/instance/governance/support-targets.yml` | Requires runtime-event-ledger evidence for support/admission. | Admission tests for journal completeness and reconstruction are not explicit enough. | Support claims can outrun runtime proof. | Add support-target gate: no supported consequential tuple without valid journal/reconstruction evidence. |
| `.octon/framework/assurance/runtime/_ops/scripts/**` | Validator system exists. | No dedicated Run Journal contract validator. | Schema drift and runtime doc inconsistency can pass. | Add `validate-run-journal-contracts.sh` and wire into architecture conformance. |
