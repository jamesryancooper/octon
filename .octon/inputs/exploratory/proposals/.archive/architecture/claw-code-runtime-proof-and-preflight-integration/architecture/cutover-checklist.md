# Cutover Checklist

## Pre-merge
- [ ] Confirm packet scope remains inside the current admitted live support universe
- [ ] Confirm no new top-level categories are introduced
- [ ] Confirm new instance policy files fit enabled overlay-compatible governance policy surfaces
- [ ] Confirm workflow manifest/registry updates are included
- [ ] Confirm assurance suites exist for each selected capability cluster

## Merge-time
- [ ] Merge authored scenario/policy/workflow files together rather than piecemeal
- [ ] Ensure `agent-led-happy-path` references the new doctor workflow before claiming onboarding closeout
- [ ] Ensure repo-consequential task workflows reference the freshness preflight before claiming broad-verification protection
- [ ] Ensure repo-shell adapter and policy-interface edits land in the same branch as the repo-shell execution-class policy

## Post-merge validation
- [ ] Run first full validation pass
- [ ] Retain all required receipts/evidence
- [ ] Run second full validation pass
- [ ] Verify no new blockers appeared between passes
- [ ] Verify generated operator digests remain derived-only
- [ ] Record closure readiness
