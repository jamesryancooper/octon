# Migration Templates

This directory contains templates used by profile-governed migration policy.

## Templates

- `template.clean-break-migration.md`
- `release-notes.template.md`

## Canonical Destination

Migration plans generated from this template must be stored in:

- `/.octon/cognition/runtime/migrations/<YYYY-MM-DD>-<slug>/plan.md`

Migration evidence generated by those plans must be stored as bundles in:

- `/.octon/output/reports/migrations/<YYYY-MM-DD>-<slug>/`
- required files:
  - `bundle.yml`
  - `evidence.md`
  - `commands.md`
  - `validation.md`
  - `inventory.md`

## Required Plan Sections

Plans must include:

1. `Profile Selection Receipt`
2. `Implementation Plan`
3. `Impact Map (code, tests, docs, contracts)`
4. `Compliance Receipt`
5. `Exceptions/Escalations`
