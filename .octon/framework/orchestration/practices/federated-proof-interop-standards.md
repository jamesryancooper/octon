# Federated Proof Interop Standards

This compatibility standard is retained for proposal lineage naming. The live
selected v6 interop rule is defined in
`portable-proof-interop-standards.md`: exchange proof as evidence, never as
authority.

Federated proof exchange remains bounded to local verification, local
acceptance, revocation, expiry, redaction, and run-authorization preservation.
Full federation compacts and delegated authority runtime are deferred from this
selected migration.
