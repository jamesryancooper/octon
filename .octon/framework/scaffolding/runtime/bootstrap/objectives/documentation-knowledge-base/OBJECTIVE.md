# Objective: {{OBJECTIVE_LABEL}}

## Workspace Goal

Use Octon in `{{REPO_NAME}}` to build, maintain, and improve documentation and knowledge assets that stay accurate, navigable, and easy to extend.

## What Octon Should Optimize For

- clarity, accuracy, and discoverability of written content
- consistent structure across guides, references, and supporting artifacts
- safe, incremental improvements to the documentation system over time

## In Scope

- docs, guides, references, playbooks, and supporting examples in this workspace
- doc structure, cross-linking, navigation, and editorial consistency improvements
- audits that identify stale, missing, duplicated, or weak documentation

## Out of Scope

- product behavior changes unless explicitly requested
- unsupported claims about system behavior, policy, or readiness
- destructive content removals without reviewable evidence or approval

## Success Signals

- readers can find and use the right material without manual guidance
- docs match current behavior closely enough to support real work
- gaps, stale content, and ambiguity are surfaced with clear next steps

## Initial Focus

- identify the primary doc sets and their intended audiences
- capture the biggest accuracy or structure gaps
- improve the most-used paths before expanding the surface area
