# Phases

## Phase 1: Findings Intake

Read the charter and the findings source. Normalize issues into:

- framing/alignment issues,
- normative integrity issues,
- authority/accountability issues,
- enforceability issues,
- terminology issues,
- dependency or precedence issues.

## Phase 2: Governance Receipt

Determine:

- release state,
- downtime tolerance,
- coordination requirements,
- migration or coexistence needs,
- rollback mechanism,
- blast radius,
- compliance constraints.

Then select exactly one `change_profile`.

## Phase 3: Target State

Define the properties the revised charter must satisfy:

- standalone clarity,
- deterministic authority,
- testable normative clauses,
- measurable success signals,
- bounded dependency model.

## Phase 4: Change Bundling

Group changes into the smallest implementation bundles that still preserve internal coherence. Prefer behavior bundles over file-by-file inventories.

## Phase 5: Verification Design

Translate the findings and target state into acceptance checks, scenario tests, and residual-risk notes.
