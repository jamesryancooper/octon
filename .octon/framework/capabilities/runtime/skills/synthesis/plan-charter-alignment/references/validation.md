# Validation

## Acceptance Criteria

- Plan contains all five required top-level sections.
- Exactly one `change_profile` is selected.
- Release-state reasoning and hard-gate facts are explicit.
- Every High and Medium audit finding is mapped to a planned change, validation check, or explicit escalation.
- Planned changes are specific enough for direct implementation of the charter file.
- Test scenarios cover framing, authority, normative integrity, terminology, dependency resilience, and precedence conflicts.
- Assumptions are explicit and bounded.

## Quality Gate

The plan is complete only when another engineer or agent could implement it without making material governance decisions that the plan failed to settle.
