# Architecture Revision Packet: Design Revised Architecture

Design the narrowest correct architectural revision that makes the selected
concepts integrable without changing Octon's constitutional kernel.

## Shared Contracts

- apply `/.octon/inputs/additive/extensions/octon-concept-integration/prompts/shared/repository-grounding.md`
- apply `/.octon/inputs/additive/extensions/octon-concept-integration/prompts/shared/architecture-revision-contract.md`

## Output

Produce a revised architecture design that:

- defines the preferred change path and narrower fallback where needed,
- identifies affected framework, instance, validator, publication, and
  migration surfaces,
- explains why the revision remains inside the current kernel,
- and stops with an escalation note instead of bluffing if a kernel challenge
  is actually required.
