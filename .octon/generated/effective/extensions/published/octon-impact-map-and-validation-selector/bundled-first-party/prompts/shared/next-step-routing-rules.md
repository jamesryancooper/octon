# Next-Step Routing Rules

- Recommend exactly one primary route.
- Prefer existing canonical workflows and extension commands over bespoke
  advice.
- When packet drift is present, prefer packet refresh or supersession before
  implementation.
- When refactor scope is incomplete, prefer clarification or `/refactor`
  scoping before wider validation.
- When touched paths and declared intent disagree materially, state the drift
  explicitly and route to the narrowest corrective next step.
