# Closeout Proposal Program

Verify every child is implemented, archived, rejected, superseded, or
explicitly deferred with rationale; follow child closeout order; retain
aggregate evidence; verify durable targets have no parent or child proposal
path dependency; regenerate registry only when safe; archive the parent only
after child lifecycle states are coherent.

For PR, CI, review, merge, branch cleanup, and sync behavior, defer to the
shared Git/worktree autonomy contract and do not create program-specific
GitHub policy. Program closeout is incomplete while any required child state,
aggregate evidence, final hygiene, review, required-check, merge, cleanup, or
sync gate remains unresolved unless the outcome is explicitly blocked or
deferred.
