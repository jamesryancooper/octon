# Repository Grounding

Every route must inspect the live repository before producing or executing a
packet-specific artifact. Required grounding includes repo ingress, the
constitutional kernel, proposal workspace rules, applicable subtype standards,
validators, promotion targets, current worktree state, and relevant generated
effective publication surfaces.

For architecture proposal creation and architecture-facing support prompts,
minimum grounding also includes the live entry, ingress, constitution,
architecture, terminology, runtime spec, runtime crate, validation script,
governance, orchestration, control-state, evidence-state, continuity,
generated, and inputs surfaces that materially affect the packet's target.
The route may inspect a narrower subset only when it records why the requested
packet is scoped away from the omitted surfaces.

If live repository state contradicts source context, the route must stop for
packet revision or emit an explicit stale-source finding instead of preserving
obsolete framing.
