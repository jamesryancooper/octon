Welcome to the Harmony handbook. This collection gathers the principles, checklists, and templates the team uses to plan, ship, and continually improve projects across the workspace.

## What's Inside

- [Frontend Framework Decision Guide](./frontend-framework-decision-guide/) — choose the right surface (Next.js or Astro) for new experiences.
- [Methodology Guides](./methodology/) — Lean AI practices, definition-of-done templates, postmortem checklists, and more.
- [Skills Playbooks](./skills/) — repeatable steps for configuring AI skills, including BMAD integrations.

Each document lives in Markdown to keep contributions lightweight. If you build on a template, copy it into your project (for example, the `spec-one-page` or `postmortem` templates) and adjust the frontmatter to capture context.

## Contributing Updates

1. Create a new Markdown file in this folder or the appropriate subdirectory.
2. Add a short frontmatter block with a `title` and `description` so the page renders cleanly in Starlight.
3. Update cross-links here or within related docs so the navigation stays discoverable.

The sidebar now links to this landing page and autogenerates the rest of the handbook structure, so new files will appear automatically without extra configuration.
