# Step 2: Classify Breaches and Hotspots

Interpret the collected latency evidence.

## Actions

1. Read the summary JSON before the Markdown report.
2. Classify the overall status as `healthy`, `watch`, or `breach`.
3. Identify:
   - required-path timing posture
   - the slowest workflows
   - the slowest steps
   - duplicate-work candidates
4. Rank findings by operational value for CI tightening.

## Output

- A prioritized interpretation of latency breaches and tightening opportunities.
