# Governance Exceptions

Normative governance contracts for waiver and exception semantics.

## Purpose

- Keep waiver and exception taxonomy in the governance surface.
- Provide a single canonical contract for override semantics under deny-by-default.
- Ensure principles and policy contracts reference one stable SSOT.

## Canonical Index

- `index.yml` - machine-readable discovery index for governance exceptions.

## Canonical Contract

- `waivers-and-exceptions.md` - canonical taxonomy and required fields.
- `principles-charter-overrides.md` - append-only override ledger for direct
  edits to `principles.md` under explicit human authorization.
