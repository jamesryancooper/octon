# Release Runbook

## Pre-Release

- Validate harness structure.
- Run engine runtime validation and targeted tests.
- Confirm migration artifacts and banlist updates for breaking changes.
- Confirm release automation is healthy:
  - `.github/workflows/release-please.yml`
  - `release-please-config.json`
  - `.release-please-manifest.json`

## Release

- Publish from the canonical engine runtime path.
- Attach evidence artifacts for build/test/gate outputs.
- Merge the release PR generated by `release-please`.
- Tag and GitHub Release are created by `release-please`; runtime binaries are
  published by `.github/workflows/runtime-binaries.yml` on the new tag.

## Credentials

- Release automation uses `AUTONOMY_PAT` when available via
  `.github/workflows/release-please.yml`.
- Minimal fine-grained PAT permissions for release acceleration:
  - `Contents: Read and write`
  - `Pull requests: Read and write`
  - `Issues: Read and write`
- See `.harmony/agency/practices/github-autonomy-runbook.md` for shared
  autonomy credential policy and validation commands.
