# Templates

Canonical reusable harness and content templates.

Bootstrap assets for `/init` live under `../bootstrap/`.

## Included

- `harmony/` — base `.harmony/` harness template
- `harmony-docs/` — documentation-focused harness variant
- `harmony-node-ts/` — Node.js and TypeScript harness variant
- `proposal-core/` — generic core files for standard-governed proposals
- `proposal-design-core/` — core files for design proposals
- `proposal-design-domain-runtime/` — domain/runtime design overlay
- `proposal-design-experience-product/` — experience/product design overlay
- `proposal-design-contracts/` — optional contracts overlay for design proposals
- `proposal-design-conformance/` — optional conformance overlay for design proposals
- `proposal-design-canonicalization/` — optional canonicalization overlay for design proposals
- `proposal-migration-core/` — subtype core for migration proposals
- `proposal-policy-core/` — subtype core for policy proposals
- `proposal-architecture-core/` — subtype core for architecture proposals
- `docs/` — documentation standards bundles
- `audits/` — bounded-audit templates
- `migrations/` — migration and release-note templates
- `document.md`, `adr.md`, `cursor-command.md` — reusable document scaffolds
