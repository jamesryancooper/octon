# Glossary Reference

**Required when capability:** `domain-specialized`

Domain terminology for the `synthesize-research` skill.

## Core Terms

| Term | Definition | Example |
|------|------------|---------|
| finding | A concrete observation extracted from source notes | "Team adoption increased 18% in Q4" |
| theme | A cluster of related findings grouped under one idea | "Onboarding friction" |
| contradiction | Two findings that appear to conflict | "Latency improved" vs "Latency worsened" |
| confidence | Strength of evidence supporting an insight | High / Medium / Low |
| synthesis | Structured consolidation of many notes into one narrative | Executive summary plus themed sections |

## Abbreviations

| Abbr. | Expansion | Context |
|-------|-----------|---------|
| md | markdown | Source note file format |
| src | source | Input corpus references |

## Domain Concepts

### Evidence Traceability

Every insight should point back to source files so readers can verify claims.

### Open Questions

Questions that remain unanswered after synthesis are explicitly tracked instead of guessed.
